# guide-site-main

这是一个用于归档和发布多个独立 HTML 页面的仓库。仓库本身不针对任何单一域名或具体网站，主要用于维护和存放各类静态页面资源。

## 项目简介

本仓库集中存放多个独立的 HTML 页面，这些页面可能用于信息展示、工具页面、临时内容页面等场景。每个页面可以独立访问，互不依赖。仓库结构清晰，便于管理和更新。

## 目录说明

```
guide-site-main/
├── pages/           # 存放所有独立 HTML 页面
├── assets/          # 页面使用的公共资源（样式、图片、脚本等）
├── docs/            # 文档或说明文件（可选）
└── README.md        # 本文件
```

- `pages/`：每个子文件夹或文件对应一个独立页面。
- `assets/`：存放多个页面可能共用的资源文件。
- `docs/`：可选的说明文档或使用指南。

## 页面归档说明

仓库中的 HTML 页面均以独立文件形式归档。每个页面通常包含完整的 HTML 结构，可直接在浏览器中打开。页面之间没有强耦合关系，新增或删除某个页面不会影响其他页面。

归档时建议遵循以下规范：
- 页面文件名使用英文小写，单词间用短横线连接（如 `example-page.html`）。
- 每个页面可附带独立的资源文件夹（如图片、样式），放在同名子目录中。
- 页面标题和内容应保持简洁，避免包含过多外部依赖。

## 维护说明

- **新增页面**：在 `pages/` 下创建新的 HTML 文件或文件夹，并确保其内容完整可访问。
- **更新页面**：直接修改对应文件内容，建议在 commit 信息中简要说明修改原因。
- **删除页面**：如需移除某个页面，请直接删除对应文件或文件夹，并确保没有其他页面引用它。
- **资源管理**：公共资源放在 `assets/` 中，页面私有资源放在页面自己的目录下。

本仓库欢迎通过 Pull Request 提交新的页面或改进建议。提交前请确保页面内容符合仓库通用归档规范，不包含任何指向特定网站或服务的链接或营销信息。

## 许可

本仓库内容仅供归档和参考使用，不构成任何形式的服务承诺。具体页面的使用请参考其自身说明。
